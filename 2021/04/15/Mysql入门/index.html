MySQL概述

mysql是关系型数据库由于其体积小、速度快、总体拥有成本低，尤其是开放源码这一特点，一般中小型网站的开发都选择 MySQL 作为网站数据库。

MySQL 数据类型

MySQL中定义数据字段的类型对你数据库的优化是非常重要的。

MySQL支持多种类型，大致可以分为三类：数值、日期/时间和字符串(字符)类型。

数值类型

MySQL支持所有标准SQL数值数据类型。

这些类型包括严格数值数据类型(INTEGER、SMALLINT、DECIMAL和NUMERIC)，以及近似数值数据类型(FLOAT、REAL和DOUBLE PRECISION)。

关键字INT是INTEGER的同义词，关键字DEC是DECIMAL的同义词。

BIT数据类型保存位字段值，并且支持MyISAM、MEMORY、InnoDB和BDB表。

作为SQL标准的扩展，MySQL也支持整数类型TINYINT、MEDIUMINT和BIGINT。下面的表显示了需要的每个整数类型的存储和范围。

日期和时间类型

表示时间值的日期和时间类型为DATETIME、DATE、TIMESTAMP、TIME和YEAR。

每个时间类型有一个有效值范围和一个"零"值，当指定不合法的MySQL不能表示的值时使用"零"值。

TIMESTAMP类型有专有的自动更新特性，将在后面描述。

约束

主键约束

主键，又称主码，是表中一列或多列的组合。主键约束(PRIMARY KEY CONSTRAINT)要求主键列的

数据唯一，并且 不允许为空。主键能够唯一表示表中的一条记录，可以结合外键来定义不同数据表之间的关系，并且可以加快数据查询的速度。主键和记录之间的关系如同身份证和人之间的关系，他们之间一一对应。主键主要分为两种： 单字段主键和 多字段联合主键。

  • 格式[字段名] [数据类型] primary key

    create table u1(
	id int primary key,
    name varchar(20)

)engine=innodb default charset=utf8

mysql> insert into u1 values(1,'zhan');
Query OK, 1 row affected (0.01 sec)

mysql> insert into u1 values(1,'zhan');
-- 约束效果 不能重复 不能为空
ERROR 1062 (23000): Duplicate entry '1' for key 'PRIMARY'
mysql> insert into u1 values(null,'zhan');
ERROR 1048 (23000): Column 'id' cannot be null

    多字段联合主键

  • 格式primary key(字段名,字段名~);

create table u2(
	`id` int,
    `name` varchar(20),
    `password` varchar(20),
    primary key(id,name)
)engine = innodb default charset=utf8;

insert into u2 values(1,'zhan','123');
insert into u2 values(1,'zha','123');
-- 两个都重复才了会报错
insert into u2 values(1,'zha','213')

mysql> insert into u2 values(1,'zha','213');
ERROR 1062 (23000): Duplicate entry '1-zha' for key 'PRIMARY'

  • 自增约束

    create table u3(
	`id` int primary key auto_increment,
    `name` varchar(20)
)engine = innodb default charset=utf8;

insert into u3 values(1,'zhan');
-- 指定字段
insert into u3(name) values('zhan');

  • 建表后的增加和删除主键

    -- 增加在字段约束
alter table [表名] add primary key(字段名)
-- 删除字段约束
alter table [表名] drop primary key;
-- 使用modify 修改字段,增加约束
alter table [表名] modify [字段名] [数据类型] primary key;

基本查询

#1. 下面的语句是否可以执行成功
select last_name , job_id , salary as sal
from employees;
#2. 下面的语句是否可以执行成功 
select * from employees;
#3. 找出下面语句中的错误
select employee_id , last_name,
salary * 12 "ANNUAL SALARY"
from employees;
#4. 显示表 departments 的结构，并查询其中的全部数据
desc departments;
#5. 显示出表 employees 中的全部 job_id（不能重复）
select distinct `job_id` from `employees`
#6. 显示出表 employees 的全部列，各个列之间用逗号连接，列头显示成 OUT_PUT
select CONCAT (`first_name`,',',`last_name`,',',IFNULL(`commission_pct`,0)) as out_put
from `employees`

  • 查询单个字段

    • select 字段名 from 表名

  • 查询多个字段

    • select 字段名 字段名 from 表名

  • 查询所有字段

    • select * from

  • 查询常量

    • select 常量

  • 查询函数

    select 函数名(实参列表)

  • 起别名

    • as  或者 空格

  • 去重

    • select distinct from 表名

  • 作用: 做加法运算

  • select 数值+数值直接运算

  • select 数组+字符 字符装换为数值再运算,转换不了就等于0

  • select null+值 结果都为null

  • concat函数

    • 可以拼接字符
    • select concat (字符1,字符2,~ );

  • ifnull

    • 判断某字段或表达式是否为null,如果为null就放回指定的值,不为null就返回原本的值
    • select ifnull (字段名,0) from 表名;

  • isnull

    • 判断某字段或表达式是否为null是就返回1,不是就为0

条件查询

  1. 语法
    1. select 查询列表 from 表名 where 条件
  2. 筛选条件分类
    1. 简单条件运算符
      1. < <= > >= <> != <=>(安全等于)
    2. 逻辑运算符
      1. &&(and) ||(or) !(not)
    3. 模糊查询
      1. like: 一般和通配符一起使用,
      2. between and
      3. in
      4. is null / is not null
  3. is null 和 (<=>)安全等于的区别
    1. is null 不能判断普通类型的数值 null值可以
    2. <=> 普通类型可以 null值也可以
